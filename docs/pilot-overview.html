三层架构

Gateway / Agent Runtime / Tool Calling

把 agent 系统拆成 3 层。每一层都有自己的 adapter 接口——Pilot 提供原生实现，但 OpenClaw / Hermes / MCP 等可以作为可选替换挂进来。

02

六阶段流水线

Layer 2 的实现

任何请求走完这 6 个阶段。每个阶段都是一个可观察、可注入 policy / hook 的方块。每一步都可以打开看、可替换实现、可以加 HITL。

Stage 1
策略 Gate
拦截高风险任务、过滤敏感关键词、检查路径黑名单
policy: keyword + path denylist
Stage 2
规划器
把任务拆步骤（单步 / CoT / ToT / ReAct）
policy: single-step (override 可换)
Stage 3
上下文检索
加载 CLAUDE.md / AGENTS.md / 历史 / RAG
v0.4.2 auto-discover 已实装
Stage 4
工具选择
从启用的工具池按 policy 选本次调用
v0.4.2 read-only inventory；v0.4.3 UI
Stage 5
执行器
沙箱运行 tool、HITL 节点、重试与错误恢复
sandbox: none；v0.5+ 可挂 worktree / container
Stage 6
校验器
跑测试 / 审计 / 用户确认 / 失败反思 prompt
trust-model；v0.5+ Hermes errorReflect

※ 最终输出阶段（第 7 段 Output Formatter）由 ReAct loop 隐含控制，未单列。

七槽配置

AgentSystem 的可编辑面

每个 AgentSystem 由 7 个槽组成。每个槽独立可编辑、可启停、可替换——借鉴 Hermes 的"分槽注入"，每面都不互相耦合。

slot 01

Instruction

多版本系统提示，可热切换。Role / Boundaries / Format / Prohibitions 分层。

slot 02

Guardrails

路径 / 命令 / 敏感数据黑名单，输出脱敏规则。

deny_path deny_cmd redact
slot 03

Workflow

执行步骤与 HITL 节点。强制先搜索再分析再验证。

steps approval_at
slot 04

Memory

多层记忆 + 压缩策略。working / episode / semantic。

tier compression
slot 05

Skills

稳定任务封装：写报告 / 处理表格 / 生成 PPT 等可复用单元。

example +
slot 06

Tools

工具白/黑名单 + policy + few-shot。详见下方开关面板。

bash read edit +
slot 07

Reasoning

scratch_pad 模板：plan → act → observe → reflect。失败反思 prompt。

plan-act reflect
meta

Self-Iteration

v0.7+ meta-agent。用户一句话，agent 自己改自己的 7 槽配置。

propose dryrun apply

06

自我迭代

v0.7+ meta-agent

你只说一句话。Pilot 的 meta-agent 自己读 session、定位要改的槽、提议改动、跑 dry-run、apply——你只需 approve 或 revert。

一个例子：「以后不要用 bash 改 .ts 文件」

用户说一句话 → interpret → locate slot → propose diff → dry-run eval → user approve → apply

找到 Tool Selector + Executor 阶段 → 提议 policy 改动 → dry-run 跑 N 个 session → 等 approve → 写到 Capability.lessons。
