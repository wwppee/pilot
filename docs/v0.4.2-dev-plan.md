# v0.4.2 详细开发计划 — 以 Pi 实际能力为锚

> 单一交付：让 Pilot 能"看见" Pi 的运行状态（manifest / sessions / tools / context）。
> 之前文档里飘的内容已经清掉。下面所有改动都用 Pi 真有的数据。

## 当前状态（verified）

- 共有 `src/` 13 个 core 模块 + 9 个 commands / 9 web routes，约 3500 行 TS
- 已实现：`stats.ts` 已经在流式聚合 sessions，但**不读 usage**
- 已实现：`jsonl-parser.ts` 已经能解析 JSONL 树状结构，但**只 preview 文本，不知道 tool calls 的成败**
- 已实现：`profile.ts` 已有 TOML CRUD，但**还没联到 pi 的 settings**

## v0.4.2 范围（5 个改动）

### 1. Token & cost 聚合（接 `stats.ts` 已经读到的 JSONL）

**为什么**：用户最常问的就是"这个月花了多少"。usage 字段已经在 JSONL 里，只是 Pilot 没读。

**改动**：
- `src/core/usage.ts` (新文件, ~120 行)
  - `aggregateUsage(filePaths: string[]): Promise<UsageReport>`
  - 流式遍历每个 JSONL 文件，提取 `AssistantMessage.usage`
  - 输出：`{ totalTokens, byModel: [{ model, input, output, cost }], byDay, byProfile }`
- `src/core/stats.ts` 调用上面，输出新增 `usage?: UsageReport`
- Zod schema 验证 `Usage` 字段存在（Pi v3 format）

**测试**：
- 单元：hand-crafted JSONL fixture 含 AssistantMessage.usage，断言聚合结果
- 集成：`pilot stats` 在 `/tmp/test-sessions/` 下，含 3 个 fixture session

**Web UI**：`/dashboard` 现有 dashboard 加入 cost / token 图表（Chart.js）

### 2. Tool call trace（从 JSONL `type: "message"` + `role: "toolResult"` 抽出）

**为什么**："哪个 tool 用了多少次、有多少失败"是用户最常问的第二问。

**改动**：
- `src/core/tool-trace.ts` (新文件, ~100 行)
  - `traceToolCalls(sessionPath: string): AsyncIterable<ToolCallEvent>`
  - 每个 `ToolResultMessage` → 一条 `ToolCallEvent { toolCallId, toolName, isError, content preview, timestamp }`
- `src/commands/session.ts` 加 `pilot session trace <id> [--tool <name>]`
- 升级 `src/core/jsonl-parser.ts` 的 `extractPreview` —— 工具结果预览要保留 isError / exitCode

**测试**：
- 单元：fixture 含 toolResult 含 `isError=true`，断言 trace 标注 failed

### 3. Project Context auto-discover（CLAUDE.md / AGENTS.md 等）

**为什么**：你最初提"Claude.md"诉求的实际入口——我们要知道 Pi 现在加载了哪些 context 文件。

**改动**：
- `src/core/project-context.ts` (新文件, ~120 行)
  - `discoverProjectContext(cwd: string): Promise<ProjectContextRef[]>`
  - 从 `cwd` 向上找 `AGENTS.md` / `CLAUDE.md` / `README.md` / `.pi/AGENTS.md` / `.cursor/rules` / `CONTRIBUTING.md` / `package.json` 的 `pi` 字段
  - 每条：`{ path, relativePath, bytes, mtime, contentPreview (前 200 字符) }`
  - 优先级按 Pi 实际加载顺序：cwd > `.pi/` > 上级目录 > HOME
- `src/commands/context.ts` (新文件, ~70 行)
  - `pilot context ls [cwd]` / `pilot context show <name>`
- Web UI：`/context` 列表页面

**注**：Pi 自己已经会加载这些文件，Pilot 只是**显式呈现**。Pilot **不在 v0.4.2 修改 Pi 行为**——那是 v0.4.3 的事（通过产出一个 "pilot-context" extension 在 `before_agent_start` 里动态改文件加载列表）。

**测试**：
- 单元：在 fixture 目录树（含 AGENTS.md / .pi/AGENTS.md / 上级 AGENTS.md / README.md）跑 discover，断言只返回最近的、跨过的不覆盖同级

### 4. Tool inventory（read-only）

**为什么**：让用户看见 Pi 现在能用哪些 tool——这是"管理 plane"的第一步，不能动。

**改动**：
- `src/core/tool-inventory.ts` (新文件, ~140 行)
  - `listToolInventory(home?: string): Promise<ToolInventoryItem[]>`
  - 来源：
    - 7 个 built-in（read/bash/edit/write/grep/find/ls）+ 每条的 safety policy
    - 从 `~/.pi/agent/extensions/*.ts` 解析 `pi.registerTool()` 调用 —— 不实际执行，用 AST 扫描找 `name: "..."` 字面量
    - 从 `~/.pi/agent/npm/package.json` 列出 npm-installed extensions
  - 每条：`{ name, source: "built-in" | "extension" | "npm", safety, description, firstSeenAt, callCount }`
- `src/commands/tool-list.ts` (新文件, ~80 行)
  - `pilot tool ls [--source built-in|extension|npm]`
  - `pilot tool inspect <name>`

**测试**：
- 单元：scan 一个真实 extension 文件（如 pi 内置的扩展），断言至少提取出 1 个工具名 + description
- 单元：npm-only scenario，返回 empty extend list 但列出 npm package

**Web UI**：`/tools` 列表页面 + `/tools/<name>` 详情

### 5. 把以上都接到 PilotService，统一 surface

**改动**：
- `src/core/service.ts` 接口加：
  ```typescript
  aggregateUsage(opts: { range: StatsRange, home?: string }): Promise<UsageReport>
  listToolInventory(opts: { home?: string }): Promise<ToolInventoryItem[]>
  discoverProjectContext(opts: { cwd: string }): Promise<ProjectContextRef[]>
  traceToolCalls(opts: { sessionId: string, filter?: { toolName?: string } }): AsyncIterable<ToolCallEvent>
  ```
- `src/core/service-impl.ts` 注入上面 4 个模块
- `src/server.ts` (Fastify) 加对应 GET endpoint：
  - `GET /usage?range=today|7d|30d`
  - `GET /tools` / `GET /tools/:name`
  - `GET /context?cwd=...`
  - `GET /sessions/:id/trace`

**Web UI**：3 个新页面（`/usage`, `/tools`, `/context`）+ `/sessions/[id]` 详情页加 trace tab
